Apply Assist

Apply Assist runs inside your normal Chrome or Firefox via the Jobsmith browser extension. When you click "Apply Assist" on a job, the extension injects a sidebar with your tailored resume, cover letter, and pre-filled answers right into the live ATS page. No separate browser is launched.

Install the extension

Save the extension to your Downloads folder, then load it in your browser.

Chrome / Edge / Brave (permanent):

  1. Get for Chrome saves the unpacked extension folder (jobsmith-extension-chrome) to your Downloads and reveals it. Chrome loads it from disk, so keep the folder around (move it somewhere permanent if you like).
  2. Open chrome://extensions, enable Developer mode (top-right).
  3. Click Load unpacked and pick that folder. The extension persists across restarts.
  4. Open the extension popup and paste the token below.

Firefox:

  1. Get for Firefox saves a Mozilla-signed jobsmith-assist.xpi to your Downloads when one has been built. Install it via about:addons → the gear icon ⚙ → Install Add-on From File… — it persists across restarts.
  2. If no signed .xpi exists yet, the unpacked folder is saved instead. Load it via about:debugging → This Firefox → Load Temporary Add-on… → pick its manifest.json. Firefox clears temporary add-ons on restart — see extension/README.md to build a signed .xpi.
  3. Open the extension popup and paste the token below.

Extension token

The extension authenticates to your local Jobsmith instance with this token. Paste it into the extension popup's "Extension token" field after install.

Folder Sync

Two-way sync of jobs, applications, and your profile with the Jobsmith iOS app through a shared folder (iCloud Drive, Dropbox, etc.). No account and no server — each device reads and writes the same folder. Your ATS login passwords are never written to the folder.

Pick the same folder you use on your iPhone. On iCloud Drive it lives in iCloud Drive (e.g. a JobsmithSync folder you create there).
Shown to your other devices so you can tell which one made a change.
How often this device checks the shared folder for changes and pushes its own. Lower is more responsive; higher is lighter on the folder provider.
When your phone can't finish scoring (you left the app, or your AI server isn't reachable from it), it leaves a request in the sync folder. With this on, this computer picks up the request, scores the remaining jobs with its own AI connection, and syncs the scores back. Off by default — turn it on if you want this machine to do the heavy lifting.
Pick which groups of settings ride the same folder. A group only syncs when it's on here and on your other device. Turn on a group on both devices to share it.
AI Connection includes your AI API key — it travels in your sync folder with the endpoint and model choices, so a phone off the inference box works without re-entering anything. It is still hidden from off-machine API callers. Never synced: ATS/Workday login passwords, the LinkedIn cookie, Adzuna/USAJobs/BLS keys, the extension token, server ports, and the sync folder itself.
